What it does

Zero-latency commands

Real shell commands run exactly as if flow weren't installed — the classification is offline and local, never touches the network.

Natural language → agent

Plain language is handed to an inline agent that can run a command, do multi-step work, or just answer — streaming its thinking and output.

Your text is preserved

flow never silently rewrites your input line. What you typed stays where you typed it; everything else prints below.

Permission you control

Reads run freely; side-effects are gated by a review level (focused / strict / yolo). Approve with y/n/a/s. Pressing a persists for the session — no repeated prompts.

Multi-provider config

~/.flow/settings.json supports Anthropic and OpenAI-compatible providers. Falls back to ~/.claude/settings.json. No SDK — raw HTTP/JSON + SSE.

Never bricks your shell

If the daemon is down or slow, flow degrades to plain zsh. Misclassified input is caught by command_not_found_handler and routed back to the agent.

Conversation per window

Each terminal window is one continuous conversation. Ask, get an answer, ask a follow-up — the agent remembers the earlier turns in that window automatically. Open a new window for a clean slate. 

~/proj 一加一等于几？
  dir ~/proj · review focused
  2

~/proj 然后平方
  dir ~/proj · review focused
  4  # ← remembers the 2 from the previous turn

flowtmp

Make a fresh temp dir and cd into it — a scratch space for the current window.

flowrsm

Continue a previous window's conversation here. Shows an arrow-key picker (each row: directory + that conversation's last message); pick one and it carries on in this window. flowrsm <id> resumes a specific one.

flowclear

Start this window's conversation over (clears its transcript and review level). Use FLOW_FRESH=1 to make a single turn ignore history.

Transcripts live under $TMPDIR/flow-<uid>/sessions/ (owner-only, cleared on reboot), keyed by an exported FLOW_SESSION_ID that survives exec zsh.

Configure a provider

Highest-priority config file: ~/.flow/settings.json — structured, supports both Anthropic and OpenAI-compatible providers: 

# ~/.flow/settings.json
{
  "provider": "anthropic",
  "base_url": "https://api.worldbase.ai",
  "auth_token": "sk-...",
  "model": "claude-opus-4-6"
}

Falls back to process env, then ~/.claude/settings.json (Claude Code compat). The discovered model is cached and shown on the right of your prompt. Check status with flow-doctor.

Activate exec zsh  (new terminals pick it up automatically)
Uninstall make uninstall  (or remove the # >>> flow >>> block from ~/.zshrc)
